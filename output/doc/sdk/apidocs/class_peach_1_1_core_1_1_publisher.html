Detailed Description

Publishers are I/O interfaces called by actions in the state model. They perform stream or call based interactions with external entities.

Publishers are I/O interfaces for Peach. They glue the actions in a state model to the target interface. Publishers can be stream based such as files or sockets, and also call based like COM and shared libraries. They can also be hybrids using both stream and call based methods to make more complex publishers.

Multiple publishers can be used in a single state model to allow for more complex operations such as writing to the registry and then calling an RPC method.

Custom publishers should implement the methods prefixed with "On" such as OnStart, OnOpen and OnInput. Additionally they should the Stream interface as needed if they will support the input action.

◆ OnInput()

virtual void Peach.Core.Publisher.OnInput ( )
protectedvirtual

Read data

This method can be overriden by custom Publishers.

The following chain of method calls will occour when an input action is performed:

  1. OnInput() is called when an input action occurs. For publishers that can read a complete peice of data with out any indication of how much data must be read can do so at this time. An example of this is UDP based protocols in which an entire packet can be read. Typically stream based publishers cannot read data at this time and should instead implement WantBytes.

  2. Cracking of input data model will commence resulting in the following method calls:

    1. WantBytes(count) is called when the cracker needs additional data. Custom publishers should perform blocking reads here.
    2. read() is called on the stream interface as needed to read from buffered data.
    3. These steps repeat until cracking has completed or no more data can be read.
  3. Input action has completed.

OnOutput(BitwiseStream)
