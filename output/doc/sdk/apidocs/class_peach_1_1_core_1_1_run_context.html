◆ controlIteration

bool Peach.Core.RunContext.controlIteration = false

Is this a control iteration. Control iterations are used to verify the system can still reliably fuzz and are performed with out any mutations applied.

The first iteration is a special control iteration. We also perform control iterations after we have collected a fault.

In later version we will likely inject control iterations every N iterations where N is >= 100.

◆ reproducingFault

bool Peach.Core.RunContext.reproducingFault = false

True when we have found a fault and are in the process of reproducing it.

Many times, especially with network fuzzing, the iteration we detect a fault on is not the correct iteration, or the fault requires multiple iterations to reproduce.

Peach will start reproducing at the current iteration count then start moving backwards until we locate the iteration causing the crash, or reach our max back search value.
